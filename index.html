1 Overview

This guide walks you through setting up Nginx as a reverse proxy with SSL/TLS termination using Let's Encrypt certificates.

What This Setup Achieves

ComponentPurpose
NginxReverse proxy, SSL termination & request routing
Let's EncryptFree SSL/TLS certificates with auto-renewal
Docker ComposeContainer orchestration for frontend & backend
KeycloakAuthentication & authorization (PKCE flow)

Traffic Flow

Internet (HTTPS :443) → Nginx (SSL Termination) → Frontend (:3000) / Backend API (:8500)

2 Verify DNS Configuration

Step 2.1: Check your server public IP

Terminal
curl -s ifconfig.me

Displays the public IP address of the current server

Step 2.2: Verify domain resolves to server IP

Terminal
nslookup dev2.marketforecaster.com

Confirms DNS A-record is correctly pointing to this server

3 Firewall Configuration

3.1 AWS Security Group (Mandatory)

TypePortSource
HTTP800.0.0.0/0
HTTPS4430.0.0.0/0

3.2 UFW Firewall (If Enabled)

Allow HTTP
sudo ufw allow 80/tcp

Allows HTTP traffic for SSL certificate verification

Allow HTTPS
sudo ufw allow 443/tcp

Allows secure HTTPS traffic

Check Status
sudo ufw status

Confirms firewall rules are active

4 Install Nginx and Certbot

Step 4.1: Update package list

Terminal
sudo apt update

Refreshes Ubuntu package index

Step 4.2: Install Nginx

Terminal
sudo apt install nginx -y

Installs Nginx web server

Step 4.3: Install Certbot

Terminal
sudo apt install certbot python3-certbot-nginx -y

Installs Let's Encrypt client with Nginx integration

Step 4.4: Verify installations

Terminal
nginx -v && certbot --version

Confirms both Nginx and Certbot are installed correctly

5 Create Nginx Configuration

Step 5.1: Create site config file

Terminal
sudo nano /etc/nginx/sites-available/dev2.marketforecaster.com

Creates a dedicated Nginx config file for this domain

📝
File name must match your domain name for clarity
⌨
Nano Editor Shortcuts:
After opening the file, use these shortcuts to paste and save:

Right-click or Ctrl+Shift+V Paste configuration
Ctrl+O Save file
Enter Confirm filename
Ctrl+X Exit nano

Step 5.2: Nginx Configuration Content

Nginx Config
server { listen 80; server_name dev2.marketforecaster.com; # Backend API routing location /api/ { rewrite ^/api/(.*) /$1 break; proxy_pass http://localhost:8500; proxy_http_version 1.1; proxy_set_header Host $host; proxy_set_header X-Real-IP $remote_addr; proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for; proxy_set_header X-Forwarded-Proto $scheme; } # Frontend routing location / { proxy_pass http://localhost:3000; proxy_set_header Upgrade $http_upgrade; proxy_set_header Connection 'upgrade'; proxy_set_header Host $host; } }

Step 5.3: Enable the site

Terminal
sudo ln -s /etc/nginx/sites-available/dev2.marketforecaster.com /etc/nginx/sites-enabled/

Creates symlink to activate the site configuration

Step 5.4: Test and reload Nginx

Terminal
sudo nginx -t && sudo systemctl reload nginx

Validates config and applies changes without downtime

6 Generate SSL Certificate

Step 6.1: Run Certbot

Terminal
sudo certbot --nginx -d dev2.marketforecaster.com

Requests and installs SSL certificate from Let's Encrypt

📧
Step 6.2: Provide your email when prompted (used for certificate expiry notifications)

7 Start Docker Containers

Step 7.1: Navigate to project directory

Terminal
cd ~/MarketForecaster/MarketForecaster-Admin-Panel

Navigate into Admin Panel project directory

Step 7.2: Build and start containers

Terminal
docker compose up -d --build

Builds images and starts containers in detached mode

8 Keycloak PKCE Configuration

Step 8.1: Verify PKCE in frontend

File: frontend/src/components/KeycloakProvider.tsx

Config Value
pkceMethod: 'S256'

Enables secure OAuth PKCE flow with SHA-256

Step 8.2: Update Keycloak Client

Add the following in Keycloak Admin Console:

Valid Redirect URIs

Keycloak
https://dev2.marketforecaster.com/*

Web Origins

Keycloak
https://dev2.marketforecaster.com
🚫
Do NOT add /* in Web Origins - it will break CORS
